Introduction Last updated: 2019-06-01

Welcome to the website documentation for Dotpot IT

This documentation provides a comprehensive guide to the structure, functionality, and features of our website. It is designed to help developers understand how the website works and how to manage it effectively.

In this documentation, you will find information about the website's design, layout, content, functionality, security, and performance. You will also find detailed instructions on how to use the website's features, such as registration, login, search, and checkout.

To help you navigate this documentation, we have organized it into different sections, each covering a specific aspect of the website. You can use the table of contents or search function to find the information you need quickly and easily.

We have also included code samples, technical specifications, and API documentation for developers who want to integrate the website with third-party tools and services.

If you have any questions or encounter any issues while using our website, please refer to our FAQs and troubleshooting guides. If you still need assistance, you can contact our support team for further help.

We are committed to providing high-quality and up-to-date documentation, so please check back regularly for updates and improvements.

Thank you for using Dotpot IT, and we hope you find this documentation helpful!

Installation

To install and set up our website, you will need to follow these general steps:

  • Clone the repository, run the following command: git@github.com:DotpotiT/website.git

Client Setup

  • Start the client-side, run the following command: cd client
  • Install the dependencies, run the following command: npm install
  • Start the server, run the following command: npm run dev
  • Open the project on browser [Server] http://localhost:port_number

API Setup

  • Start the server-side, run the following command: cd api
  • Install the dependencies, run the following command: npm install
  • Start the server, run the following command: npm start

Admin Setup

  • Start the client-side, run the following command: cd admin
  • Install the dependencies, run the following command: npm install
  • Start the server, run the following command: npm run dev
  • Open the project on browser [Server] http://localhost:port_number

APIs

API integration is an important part of modern web development, and we have used several latest technologies to integrate APIs into our website.

NodeJs

Node.js is known for its ability to handle large volumes of requests and data-intensive applications, making it a popular choice for building scalable and high-performance websites. We have implemented nodeJs to use javascript on both the server-side and client-side of the website to improve the website's responsiveness and overall user experience.

ExpressJs

We have used a popular nodeJs framework ExpressJs to develop our website. It provides a variety of features and functionalities that help simplify the development process and make it easier to build robust, scalable, and maintainable web applications.

Some of the main purposes of using Express.js in web application development include:

  • Routing: Express.js provides a simple and flexible routing system that makes it easy to define routes and handle HTTP requests. This allows developers to create RESTful APIs and handle requests for different types of resources, such as HTML pages, JSON data, or static files.
  • Middleware: Express.js middleware allows developers to add functionality to their applications, such as authentication, logging, and error handling. This helps keep the code organized and makes it easier to maintain and debug.
  • Templating: Express.js supports a variety of templating engines, such as EJS and Handlebars, which allow developers to generate HTML dynamically and build dynamic web pages.
  • Scalability: Express.js is designed to be highly scalable and can handle a large number of requests efficiently. This makes it a good choice for building high-traffic web applications.
  • Community and Ecosystem: Express.js has a large and active community of developers, which means there are many plugins, modules, and tools available that can be easily integrated into your application. This can help speed up development and add new features and functionality to your application.

Axios

Axios is a popular JavaScript library that is used for making HTTP requests from a web browser or Node.js. It provides an easy-to-use interface for making asynchronous HTTP requests, and it supports a variety of request and response types, such as JSON, XML, and FormData. We have used Axios to provide a convenient and consistent interface for making HTTP requests in our website development, while also providing additional features for error handling and response interception.

Intergrations

Integrating APIs is an important part of building a web application, as it enables the application to communicate with other systems or services. We have a collection of APIs to have a faster, responsive user interface and handle real-time communications between client and servers.

ReactJs

React is a JavaScript library used for building user interfaces. We have used React to create reusable UI components that can interact with our API endpoints. We have used a number of reusable react components, state APIs, react Hooks and context APIs to make our website fast and responsive.

MongoDB

MongoDB is a NoSQL database that provides high performance and scalability for web applications. We have used it to store data for our application, and also have used a library called Mongoose to define user schemas and models for our data.

Socket.io

Socket.io is a JavaScript library that enables real-time, bidirectional communication between clients and servers over websockets. We have used Socket.io to create a real-time chat application, where users can interact with server in real-time and get their queries solved.

Utilities

Here is a list of some common and popular utilities that we have used in our website:

npm

npm stands for "Node Package Manager." It is a package manager for the JavaScript programming language and is primarily used for managing dependencies in Node.js projects.

webpack

Webpack is a popular module bundler for JavaScript applications. It allows developers to bundle multiple modules and assets into a single file or set of files for deployment to a browser or server. Webpack offers many features, such as code splitting, hot module replacement, and tree shaking, which help optimize the performance and speed of web applications.

JWT

JWT or "JSON Web Token" is a popular open standard for creating and transmitting secure authentication tokens between parties. JWTs are used for securely transmitting information between parties, such as a client and server, without the need for session management on the server. This is done by digitally signing and encoding the information into a JSON object, which can then be sent as a compact string.
